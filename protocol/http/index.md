# http

## 前置知识

### TCP/IP 四层模型
- MAC 地址是这个在网络世界中标识每一台联网设备的唯一标识，烧录在网卡中
- TCP/IP 四层模型是当前互联网使用的模型。包含：应用层，传输层，网络互联层，网络接口层
```
应用层：你的邮件客户端生成邮件数据，并使用 SMTP 协议进行格式化。
传输层：SMTP 数据交给 TCP。TCP 将其分割，并添加 TCP 报头（包含源端口和目的端口）形成 TCP 端。
网络层：TCP段被交给 IP。IP添加 IP 包头（包含IP地址和目的IP地址），形成IP数据报。
网络接口层：IP数据报被交给以太网驱动程序。它被封装成以太网帧，添加帧头（含源MAC地址和目的MAC地址）和帧尾（CRC校验码）
物理介质：以太网帧被转成为比特流，通过网线或无线电波发送出去。
```  

### TCP
- 为了服务器安全和资源享用平等，浏览器限制在同一域名下创建并行的TCP连接数为6。
- TCP是一种可靠的、有序的字节流协议。它保证接收方收到的字节顺序与发送方发出的完全一致。
- TCP 连接的本质是在客户端和服务器的内存中各自维护一组状态变量，同一个
- TCP连接在保持连接的情况下可以发送多个请求TCP的三次握手，四次挥手

## http/1.x
- 在HTTP/1.1中，默认使用 持久连接，允许在同一个TCP 连接上发送多个请求。但是，这些请求的响应必须是按顺序返回的。这就是"先进先出”的队列。
- 当时优化方案
    1. 域名分片:开启多个TCP连接
    2. 2.资源合并:减少请求数量

## http/2.
- HTTP/2 在传输层和应用层之间引入了一个*二进制分帧层*，基于二进制分帧，HTTP/2可以在一个单一的TCP连接上并行交错地发送多个请求和响应，HTTP/2 解决了 应用层的队头阻塞
- 几乎所有主流浏览器都只支持**加密连接**上使用 HTTP/2
- 缺点是当发生TCP 数据包在网络传输中丢失或延迟，那么后面的数据包都必须等待这个丢失的包被重传并成功接收后，才能继续被处理。
- 优化策略调整:
  1. 通过雪碧图减少请求数量收益变小，HTTP/2单个 TCP 连接可以乱序连续发送请求和响应信息
  2. 域名分片:通过将资源分散到不同域名来突破浏览器连接数限制，因为需要创建新的连接而失去优势

## http/3层
为了彻底解决TCP层对头阻塞， HTTP/3 直接放弃了TCP，转而使用谷歌开发的 QUIC 协议作为传输层。

## 队头阻塞
- HTTP 层对头阻塞（同一个 TCP 链接中的请求和响应是串行的）
- TCP 层队头阻塞(网络丢包的情况下会造成 TCP层队头阻塞)
